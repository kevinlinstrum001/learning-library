Logic Gates

A lesson for Klins on how simple true-or-false decisions become circuits, how those circuits became modern computers, and how the same ideas appear on the Raspberry Pi bench today.

The central idea: a logic gate is a physical circuit that performs a rule. It receives one or more electrical inputs, interprets them as LOW or HIGH, and produces an output according to Boolean logic. Complex digital systems are built by combining enormous numbers of these simple rules.

1. Binary is a physical agreement

Digital logic begins by dividing electrical conditions into two useful states.

LOW

LOW is interpreted as 0, false, off, or inactive. It is usually represented by a voltage near ground.

HIGH

HIGH is interpreted as 1, true, on, or active. It is represented by a voltage above a defined threshold.

Thresholds matter

Real circuits do not see perfect mathematical zeros and ones. They see voltages. A logic family defines which voltage ranges count as LOW, HIGH, or uncertain.

Bench connection

When we set a Raspberry Pi GPIO pin to HIGH, software is not creating an abstract truth by itself. It is commanding transistor circuitry inside the processor to produce a voltage. That voltage can drive another logic input, an LED through a resistor, or the control side of a relay interface.

Each gate answers a different kind of question.

NAND

AND, then invert

The output is 0 only when every input is 1. NAND is a universal gate.

NOR

OR, then invert

The output is 1 only when every input is 0. NOR is also a universal gate.

XOR

Detect a difference

The output becomes 1 when the inputs are different. XOR is central to addition, comparison, and parity checks.

3. Examples of use

Logic gates turn combinations of conditions into decisions.

Safety interlock

A machine may run only when the guard is closed AND the start button is pressed.

Alarm system

An alarm may trigger if the door sensor OR the window sensor becomes active.

Mismatch detector

An XOR gate can indicate that two signals disagree, which is useful in checking data or comparing states.

Relay permission rule

The Pi may energize a relay only if a sensor is active AND the manual-enable input is true.

Two-way conditions

A light can turn on when switch A OR switch B is active, even if the switches are far apart.

Memory and state

Cross-coupled NAND or NOR gates can form a latch, allowing a circuit to remember one bit even after the original input changes.

4. How logic gates developed

The rules stayed simple while the physical machinery changed dramatically.

1840s

Boolean algebra

George Boole developed a mathematical system built around true and false relationships. At the time, it was abstract mathematics rather than electronics.

1930s

Logic meets switching circuits

Claude Shannon showed that Boolean algebra could describe relay switching networks. This connected mathematical logic to real electrical systems.

1940s

Relays and vacuum tubes

Early digital computers used electromechanical relays and then vacuum tubes to perform logic. They worked, but they were large, hot, power-hungry, and prone to failure.

1950s

Transistor logic

Transistors replaced vacuum tubes. They were smaller, faster, cooler, and more reliable, making practical digital electronics much easier to build.

1960s–1970s

Integrated circuits

Multiple transistors and gates were manufactured together on one chip. Families such as TTL became standard building blocks, including familiar packages like the 7400 series.

Today

Billions of gates on a chip

Modern processors use extremely small MOSFET transistors arranged into CMOS logic. Billions of transistors cooperate to form gates, memory, arithmetic units, communication systems, and processors.

5. How gates are achieved today

Most modern digital logic is built from MOSFET transistors using CMOS design.

Transistors act as controlled switches

A transistor uses one electrical signal to control whether current can flow through another path. Arranged carefully, transistors reproduce the truth table of a NOT, AND, OR, NAND, NOR, or XOR gate.

CMOS uses complementary devices

CMOS combines two transistor types so that one network pulls the output upward while another pulls it downward. This arrangement gives strong digital states and low static power consumption.

Software describes hardware

Engineers often design logic using hardware-description languages such as Verilog or VHDL. The design is synthesized into gate networks for FPGAs or custom chips.

Microcontrollers hide the gates

The Raspberry Pi, ESP32, and Arduino expose simple functions such as reading a pin or setting an output, but those functions are executed by vast networks of logic gates inside the processor.

The modern shift

We rarely wire thousands of physical gates by hand now. Instead, we use integrated circuits, programmable logic, microcontrollers, and software. The gate-level logic is still present, but it has been packaged into larger and more convenient layers.

6. Projects for our bench

These experiments connect the poster directly to the Raspberry Pi, ESP32, breadboard, LEDs, switches, and relays.

Two-switch AND circuit

Wire two pushbuttons so an LED turns on only when both are pressed. First build it physically, then duplicate the same rule in Python using two GPIO inputs.

Two-switch OR circuit

Build a circuit in which either switch can light the LED. Compare the hardware path with a software statement using or.

NOT gate with a transistor

Use a transistor and resistor network so that a HIGH input produces a LOW output. This makes inversion visible as an electrical action rather than only a truth table.

NAND-only challenge

Use several NAND gates from a 74HC00 chip to construct NOT, AND, and OR behavior. This demonstrates why NAND is called universal.

Relay permission controller

Let the relay activate only when a motion sensor is active AND a manual-enable switch is on. This combines sensing, logic, and control.

One-bit memory latch

Build an SR latch with NAND or NOR gates. Press one button to set the output and another to reset it. The circuit remembers its state.

7. Logic in code and logic in hardware

The same rule can exist in different physical forms.

Hardware logic

The result appears because voltages move through transistor networks. It is immediate, parallel, and does not require a program loop.

Software logic

A processor reads values, executes instructions, and calculates the result. The code appears abstract, but the processor performs every step through physical logic gates.

The larger lesson for Klins

Logic gates are where electrical engineering and reasoning meet. A condition such as “both switches must be on” can be written as Boolean algebra, expressed in Python, drawn as a gate diagram, or built from transistors. These are not separate ideas. They are different representations of the same rule.
